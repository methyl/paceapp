# wasm-fit

Rust → WASM FIT decoder. Replaces the JavaScript `fit-file-parser` and the
`@garmin/fitsdk` Decoder pass that used to dominate `parseFitFile` cost in
both tests and the browser.

The `@garmin/fitsdk` Encoder is still used for re-export; this crate emits
`rawMessages` in the exact camelCase shape its Encoder consumes (Date
timestamps, camelCased enum values, enhanced/running fields aliased to
their canonical names).

## Layout

- `src/lib.rs` — `parse_fit(bytes)` exported via `wasm-bindgen`.
- `pkg/` — generated by `wasm-pack`; checked in so consumers don't need a
  Rust toolchain.
- `loadWasm.ts` — TypeScript loader that initializes the wasm module from
  the right place under both Vite (browser) and Vitest (Node).

## Rebuild

```
npm run wasm:build
```

Requires `cargo` + `wasm-pack`. The Cargo profile disables `wasm-opt`
because hosts often can't reach the binaryen GitHub release; `lto +
opt-level=3 + strip` already gives most of the size benefit.
